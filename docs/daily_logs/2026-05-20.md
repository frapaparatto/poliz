**Architecture reasoning:**

- `display()` was removed as a pure virtual because rendering is a presentation concern and belongs in the CLI layer, not in the domain. The domain classes should expose data via getters; the controller or a dedicated formatter decides how to display it. A pure virtual `display()` on the base class would force the domain to know about output format, which violates the layered architecture.
- `updated_at_` is `protected` rather than private because the entire class hierarchy owns the concept of "this object was last modified." Appointment and Contract setters are legitimate writers of that timestamp, not external callers. Keeping it `protected` preserves encapsulation from outside code while giving derived classes direct access, consistent with how `Policy` handles the same field within a single class.
