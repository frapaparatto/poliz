**Architecture reasoning:**

- Making `save()` public for testing was deliberate and noted in ADR-021. The alternative would have been a friend declaration or a test-only subclass, both of which are worse. `save()` is a legitimate operation that controllers already call; there is no encapsulation principle violated by tests calling it directly.
- The GitHub Actions workflow uses `ctest --output-on-failure` so CI logs show the actual Catch2 output only when a test fails, keeping passing runs clean.
