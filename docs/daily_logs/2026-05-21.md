**Active branch:** feature/interactions-crud

**What I built:**

- Created `interaction_status.hpp` and `interaction_status.cpp`. Extracted enum-to-string and string-to-enum conversion for `InteractionType` and `ContractStatus` into a dedicated file. Both directions throw `std::invalid_argument` on unrecognized values.

- Wrote `IInteractionRepository` interface. Declared `insertInteraction` and `updateInteraction` as taking `std::unique_ptr<Interaction>` by value (ownership transfer, consistent with Interaction being abstract and non-copyable). All filter and find methods return `std::vector<std::unique_ptr<Interaction>>` for the same reason. `filterByType` and `filterByDate` return an empty vector on no match, not `std::optional`, because an empty result is a valid answer.

- Implemented `CsvInteractionRepository` in full:
  - `insertInteraction`, `removeInteraction`, `updateInteraction`: lock mutex, mutate `interactions_`, set `dirty_`. `removeInteraction` uses `std::remove_if` with assert to catch unknown UUIDs. `updateInteraction` uses `std::find_if` with assert, then moves the updated object in-place.
  - `findByUuid`: returns a single clone or `nullptr` if not found.
  - `findByClientUuid`: range-for over `interactions_`, clones and pushes all matching by `getClientUuid()`.
  - `findAll`: reserves capacity upfront to avoid reallocations, then clones every entry.
  - `filterByType`: range-for, clones where `getType()` matches the requested `InteractionType`.
  - `filterByDate`: range-for, clones where `getDate() >= start_date && getDate() <= end_date`. Works correctly because dates are ISO-8601 (YYYY-MM-DD), which is lexicographically ordered. No conversion needed at this layer.
  - `serializeAppointment` and `serializeContract`: write all fields as CSV using `ostringstream`. Optional fields serialized as empty string via `.value_or("")`.
  - `serialize`: dispatches via `dynamic_cast` to the correct serializer. Returns `runtime_error` if the cast fails.
  - `deserialize`: reads UUID, client UUID, type discriminator; branches to construct `Appointment` or `Contract` with full field parsing. `stoi` and `stod` wrapped in try/catch to produce readable errors on malformed CSV.
  - `load`: builds a temporary vector, swaps into `interactions_` atomically, clears `dirty_`.
  - `save`: writes to `filepath_.tmp`, then renames to the real path. Atomic on POSIX systems.

**Architecture reasoning:**

- Enum-to-string functions live in `interaction_status` rather than in `interaction.hpp` to keep the domain entity file focused on data and behavior. String conversion is I/O-adjacent: needed only by the CSV layer and display, not by domain logic.
- `serialize` uses `dynamic_cast` rather than `static_cast` when downcasting to concrete types. Both would work given the type discriminator already branched on, but `dynamic_cast` catches a mismatched type at runtime through the vtable and fails with a clear error rather than silently producing garbage. The runtime cost is negligible.
- All const query methods (`findByUuid`, `findByClientUuid`, `findAll`, `filterByType`, `filterByDate`) lock the mutex. `mtx_` is declared `mutable` to allow this from const methods. This is the standard pattern for protecting shared state in a const-correct class.
